# WebTV

WebTV - это небольшой self-hosted IPTV-плеер, который работает прямо в браузере. Добавьте один или несколько M3U-плейлистов, откройте веб-интерфейс с любого компьютера в сети и смотрите ТВ без установки отдельного плеера на каждое устройство.

Это проект just fun, написанный с помощью LLM. Мне просто захотелось иметь видеоплеер для IPTV в браузере: запустить его один раз на сервере или компьютере и открывать с любых браузеров, вместо того чтобы устанавливать IPTV-плеер на каждый компьютер.

[English version](README.md)

## Возможности

- IPTV-плеер в браузере со встроенным Vue-интерфейсом.
- Несколько M3U-плейлистов: название, включение/отключение, ручное обновление и автообновление по интервалу.
- Импорт EPG/XMLTV с текущими и будущими передачами.
- Группы каналов, поиск, избранное, скрытие групп и каналов отдельно для каждого плейлиста.
- Поддержка архива/timeshift, если плейлист содержит archive/catchup-метаданные.
- Автоматический выбор режима потока: прямой proxy, если браузер умеет воспроизвести поток, или транскодинг через ffmpeg, если не умеет.
- HLS через hls.js там, где это нужно, с fallback на нативное воспроизведение.
- Поиск и выбор аудиодорожек, настройка языка аудио по умолчанию.
- Опциональный деинтерлейс для interlaced IPTV-потоков.
- Светлая, темная и системная тема интерфейса.
- Английская и русская локализация.
- Хранение данных в одном SQLite-файле.

## Как работают воспроизведение и транскодинг

WebTV не транскодирует все подряд. Он старается сохранить поток оригинальным, если браузер может его воспроизвести:

- **Direct mode** проксирует совместимые потоки в браузер. Для HLS WebTV переписывает ссылки на сегменты внутри `.m3u8`, чтобы вложенные сегменты загружались через тот же сервер.
- **Transcode mode** запускает `ffmpeg` и отдает fragmented MP4 (`video/mp4`) для совместимости с HTML5 video.
- Сервер проверяет HLS/кодеки и включает транскодинг для кодеков, которые часто не поддерживаются браузером: HEVC/H.265, MPEG-2 video, MPEG-4 video, AC3/EAC3/A52, MP2 и MPGA.
- Если возможно, ffmpeg копирует совместимую видео- или аудиодорожку и перекодирует только несовместимую часть.
- Если в интерфейсе включен деинтерлейс, видео декодируется и кодируется заново через фильтр `yadif`.
- Если выбранная аудиодорожка не работает через браузерный/HLS-путь, интерфейс может запросить audio fallback через ffmpeg.
- Определенный режим потока кешируется для канала на `WEBTV_STREAM_MODE_CACHE_TTL_MIN` минут.
- Для прямого proxy есть idle timeout и повторные подключения, чтобы переживать нестабильные IPTV-источники.
- HLS manifest кешируется на короткое время; опциональный manifest backoff снижает частоту запросов к провайдеру, если live manifest не меняется.

Docker-образ уже содержит `ffmpeg`, `ffprobe`, CA certificates и timezone data. Если запускать бинарник напрямую, установите `ffmpeg` отдельно и убедитесь, что `ffmpeg` и `ffprobe` доступны в `PATH`, либо настройте `WEBTV_FFMPEG_BIN`.

## Настройки

Настройка выполняется через переменные окружения.

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `WEBTV_ADDR` | `:8080` | HTTP-адрес для прослушивания. Например: `:8080` или `127.0.0.1:8080`. |
| `WEBTV_DB_PATH` | `webtv.db` | Путь к файлу SQLite-базы. В Docker Compose по умолчанию используется `/data/webtv.db`, а `/data` монтируется в локальную папку `./data`. |
| `WEBTV_LOG_LEVEL` | `info` | Уровень логов: `debug`, `info`, `warn`, `error`. |
| `WEBTV_APP_TITLE` | `WebTV` | Заголовок приложения в интерфейсе. |
| `WEBTV_SINGLE_CONN_POLICY` | `reject` | Историческая настройка политики соединений для потоков. Сейчас обработка потоков допускает retry/reopen от браузера, поэтому переменная в основном оставлена для совместимости. |
| `WEBTV_FFMPEG_BIN` | `ffmpeg` | Имя или абсолютный путь к ffmpeg для транскодинга. Для анализа потоков также используется `ffprobe`. |
| `WEBTV_STREAM_IDLE_TIMEOUT_SEC` | `12` | Таймаут простоя при прямом копировании потока. Если данные не приходят за это время, WebTV переподключается. |
| `WEBTV_STREAM_RETRY_MAX` | `5` | Максимальное количество повторных подключений для зависшего direct stream. |
| `WEBTV_STREAM_MODE_CACHE_TTL_MIN` | `360` | Время кеширования определенного режима direct/transcode для канала. |
| `WEBTV_STREAM_MANIFEST_BACKOFF_ENABLED` | `false` | Включает короткий backoff для HLS manifest, если live manifest не меняется. |
| `WEBTV_IMAGE_TAG` | `latest` | Тег Docker-образа `ghcr.io/asd2003ru/webtv` для `docker-compose.yaml`. |
| `WEBTV_PORT` | `8080` | Порт на хосте, который пробрасывается на порт `8080` контейнера. |

Готовый пример есть в `.env-example`.

## Установка через Docker

Docker - основной и рекомендуемый вариант установки WebTV. В репозитории уже есть готовый `docker-compose.yaml`, поэтому достаточно скачать его и подготовить `.env` файл.

Скачайте из репозитория эти файлы:

- `docker-compose.yaml`
- `.env-example`

Затем запустите:

```bash
cp .env-example .env
docker compose up -d
```

После запуска откройте:

```text
http://localhost:8080
```

Данные хранятся в локальной папке `./data` рядом с `docker-compose.yaml`. WebTV автоматически создаст `data/webtv.db` при первом запуске.

Минимальный запуск одной командой:

```bash
docker run -d \
  --name webtv \
  --restart unless-stopped \
  -p 8080:8080 \
  -v ./data:/data \
  ghcr.io/asd2003ru/webtv:latest
```

## Установка через бинарник

Скачайте или соберите бинарник `webtv`, затем запустите:

```bash
WEBTV_ADDR=:8080 \
WEBTV_DB_PATH=./data/webtv.db \
WEBTV_FFMPEG_BIN=ffmpeg \
./webtv
```

Для запуска бинарника нужны:

- установленные `ffmpeg` и `ffprobe`;
- современный браузер на клиентских устройствах;
- доступ от хоста с WebTV до URL вашего IPTV-провайдера.

## Сборка

Собрать бинарник:

```bash
make install
make build
```

Бинарник появится здесь:

```text
bin/webtv
```

То же самое вручную:

```bash
npm --prefix web install
npm --prefix web run build
go build -o bin/webtv ./cmd/webtv
```

Запуск из исходников:

```bash
make run
```

Тесты:

```bash
make test
```

## Сборка Docker-образа

Сборка через Docker Compose:

```bash
docker compose -f docker-compose-build.yaml up -d --build
```

Или сборка образа напрямую:

```bash
docker build -t webtv:latest .
```

Dockerfile сначала собирает Vue frontend, затем встраивает его в Go-бинарник и копирует бинарник в небольшой Alpine runtime image с установленным ffmpeg.

## Как добавить новый язык интерфейса

Локализация находится в `web/src/i18n/messages.js`, а логика выбора языка - в `web/src/composables/useUILanguage.js`.

Чтобы добавить язык:

1. Добавьте новый верхнеуровневый ключ в `messages`, например `de`.
2. Скопируйте все ключи из `en` и переведите значения. Имена ключей должны остаться такими же.
3. Обновите `systemLanguageCode()` в `useUILanguage.js`, если новый язык должен определяться автоматически по `navigator.language`.
4. Разрешите новый режим языка в `currentLanguage()`, `onLanguageModeChange()` и `initLanguageMode()`.
5. Добавьте видимый вариант выбора в `web/src/components/PlaylistsPanel.vue`.
6. Добавьте label-ключ вроде `lang_german` во все языковые блоки.
7. Запустите `npm --prefix web run build` и проверьте интерфейс.

## Заметки

- Приложение рассчитано на личное/self-hosted использование.
- WebTV не предоставляет IPTV-контент. Нужны собственные легальные M3U/EPG-источники.
- Транскодинг нагружает CPU. Если несколько клиентов одновременно смотрят каналы с транскодингом, запускайте WebTV на достаточно производительном железе.
